## Как запустить проект локально

### 1. Подготовка окружения
   ```env
   DB_HOST=localhost
   DB_PORT=5432
   DB_USER=postgres
   DB_PASS=Suharik15022006
   DB_NAME=practice_crm
   ```
2. Установите зависимости проекта:
```
pip install -r requirements.txt
```

### 2. Запуск инфраструктуры (БД и Kafka/Redpanda)
```
docker compose up -d
```
`http://localhost:19082` — здесь можно наглядно просматривать топики и сообщения.

### 3. Применение миграций базы данных
```
alembic upgrade head
```

### 4. Запуск веб-сервера (API)
```
uvicorn src.main:app --reload --port 8080
```
Документация Swagger будет доступна по адресу: `http://localhost:8080/docs`

### 5. Запуск воркеров
1. **Запуск Outbox Publisher**:
   ```
   python -m src.publisher
   ```
2. **Запуск Kafka Consumer**:
   ```
   python -m src.consumer
   ```

## Тестирование и ручная отправка событий

```
python -m src.send_test_moderation
```
Скрипт предложит ввести UUID заявки, после чего сам отправит нужное событие в Kafka.

#### Пример JSON-события для ручной отправки в Kafka
```json
{
  "event_id": "d3b07384-d113-4956-a5e2-aa5898b3f289",
  "event_type": "lead_moderation_finished.v1",
  "aggregate_id": "8fa53828-5695-46c5-bf7d-2b3671a566a7",
  "occurred_at": "2026-07-06T00:35:00Z",
  "payload": {
    "lead_id": "8fa53828-5695-46c5-bf7d-2b3671a566a7",
    "approved": true,
    "reason": null
  }
}
```

#### Что сделано:
  
1. Разработано асинхронное FastAPI API с ручками создания заявки ( POST ) и получения по ID ( GET ).
2. Реализован паттерн Transactional Outbox для интеграции БД и очередей сообщений. Запись заявки и запись события в outbox-таблицу происходят одновременно в рамках одной транзакции PostgreSQL.
3. Написан воркер publisher, который раз в секунду опрашивает БД и отправляет события в топик Kafka  leads.events.v1 .
4. Написан воркер Kafka Consumer, который обрабатывает входящие решения модерации. Реализована проверка идемпотентности (защита от дубликатов сообщений) с использованием таблицы  inbound_events.
5. Настроен файл docker-compose.yml  (PostgreSQL, Redpanda в качестве Kafka-брокера и Redpanda Console).
6. Написан скрипт для ручной отправки тестовых событий в Kafka.
